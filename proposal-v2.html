6. 기술 스택 선정 근거 (Astro + Supabase vs Spring Boot)

고객 지침서에서 "국내 인력 풀이 넓고 모듈화가 용이한 Spring Boot 또는 NestJS/Node.js"를 요구하셨습니다. 아래에서 고객 조건에 맞는 객관적 비교를 제시합니다.

6.1 고객 조건 정리

  • 내부 전산 인력 없음 (향후 외부 유지보수 위탁)
  • EC2 1대 (비용 최소화)
  • 1인 관리자 유지보수 가능해야 함
  • 개인정보 취급 → 보안 필수
  • 외부 콜센터 접근 제어 필요

6.4 고객 요구 수용 — Vue/React Islands

고객의 "React 또는 Vue" 요구를 수용하여, Astro의 Islands Architecture에서 Vue 또는 React 컴포넌트를 프론트엔드 UI로 채택합니다. Astro는 Vue/React/Svelte 모두 지원하며, 고객이 향후 유지보수를 외주할 때 Vue/React 인력을 활용할 수 있습니다.
